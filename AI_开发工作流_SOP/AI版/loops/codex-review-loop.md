# codex-review-loop —— 代码预审（AI 执行版）

> **何时启动**：用户要"审 PR / code review / 预审 / 检查改动"。  
> **禁止**：罗列无 SEV 标签的建议；SEV1 > 5 条；空建议不可执行；编造行号。

---

## 输入清单（缺即停）

| 项 | 必需 | 缺失时动作 |
|---|---|---|
| Git diff | 必需 | 用 Bash 跑 `git diff main...HEAD` 或要用户提供 |
| PRD / Issue | 必需 | 反问"对应的 PRD 在哪" |
| 根 CLAUDE.md | 必需 | 缺时按 Java/Spring 通用规范 |
| 模块级 CLAUDE.md | 可选 | 缺时跳过 RC2 模块约定部分 |
| 测试报告（如有 tdd-loop 产物） | 建议 | 缺时 RC3 仅判断"是否有测试" |

---

## 流程（6 步，禁止跳步）

1. **读 diff**：识别本次改动涉及的文件、模块、功能边界
2. **逐维度查**：按 RC1–RC6 顺序检查，**每维度必须输出结果**（即使无问题也要标 ✅）
3. **打标签**：每条问题打 SEV1/SEV2/SEV3 + 置信度（高/中/低）
4. **自查**：按 PASS 条件 AC1–AC6 填自查表
5. **降级 / 合并**：若 SEV1 > 5，立即合并同类或降级到 SEV2
6. **终输出**：4 段（总览 + 问题列表 + 自查表 + 待确认清单）

---

## PASS 条件（自查标准，严格解释）

### AC1 每条问题都有位置 + 引用
- ✅ 100% 问题含 `文件:行号` + 引用 ≤ 5 行代码片段
- ⚠️ ≥ 90%
- ❌ < 90%

### AC2 每条问题都有 SEV + 置信度
- ✅ 100% 含两个字段
- ⚠️ 漏 1-2 处
- ❌ 漏 ≥ 3 处

### AC3 SEV1 数量 ≤ 5
- ✅ SEV1 ≤ 5
- ⚠️ SEV1 = 6-7（需在自查中说明为什么不合并）
- ❌ SEV1 ≥ 8（必须降级 / 合并）

### AC4 RC1–RC6 全覆盖
- ✅ 6 个维度都被检查并在自查表标 ✅ 或列出问题
- ⚠️ 漏 1 个维度
- ❌ 漏 ≥ 2 个维度

### AC5 总览"建议特别关注" ≤ 3 条
- ✅ ≤ 3 条
- ⚠️ 4 条（说明理由）
- ❌ ≥ 5 条

### AC6 不确定的判断必须降级
- ✅ 所有置信度"低"的问题 SEV 不超过 SEV2
- ⚠️ 出现 1 处置信度"低"但打 SEV1
- ❌ ≥ 2 处

---

## RC1–RC6 检查项明细

### RC1 PRD 一致性
- 每条 AC 在 diff 中是否有对应实现
- diff 中是否实现了 PRD 未要求的功能（范围蔓延）

### RC2 CLAUDE.md 约定
- 分层是否被破坏（Controller 直接调 Mapper / Service 调 Service 跨域）
- 包结构、命名是否违反约定
- 禁用清单（如禁 Lombok @Data、禁 new Date()）是否被踩

### RC3 测试覆盖与质量
- 新增 public 方法是否都有测试
- 是否引入 assertNotNull / assertTrue(true) 类恒真断言
- 是否有 @Disabled 的测试

### RC4 并发 / 事务 / 空指针 / 安全
- 共享状态是否加锁 / CAS / 队列化
- @Transactional 传播是否合理，外部调用是否在事务内
- 空指针：未检查的 get() 链、未处理的 Optional
- 安全：SQL 注入、敏感日志、密钥硬编码、CSRF / XSS（如涉及）

### RC5 命名 / 分层 / 重复代码
- 变量 / 方法 / 类命名是否一致
- 是否存在可提取的重复代码（3+ 处相似逻辑）
- 是否有死代码 / 未使用 import

### RC6 文档同步
- 新增接口是否更新 docs/entry-points.md
- 新增业务术语是否进 docs/glossary.md
- CLAUDE.md 中引用的文件 / 章节是否仍存在

---

## SEV 严重度分级（严格）

### SEV1（必须本 PR 修，否则不可合并）
- 必然导致线上事故的 bug
- 安全漏洞（SQL 注入、密钥泄露、未鉴权接口）
- 违反核心架构约束（破坏分层 + 影响其他模块）
- 数据漂移 / 资损风险

### SEV2（应在本 PR 修）
- 可能导致问题但非必然
- 违反一般约定
- 明显代码异味（空指针风险、重复代码 ≥ 5 处）
- 测试缺失（关键路径）

### SEV3（可下个 PR 跟）
- 可读性、命名
- 文档轻微落后
- 微小重构机会

### RC1 PRD 一致性
| SEV | 位置 | 问题 | 建议 | 置信度 |

或：本维度无问题 ✅

### RC2 CLAUDE.md 约定
...

### RC3 测试覆盖与质量
...

### RC4 并发 / 事务 / 空指针 / 安全
...

### RC5 命名 / 分层 / 重复代码
...

### RC6 文档同步
...

## 三、自查表
| AC | 评分 | 证据 |
|---|---|---|
| AC1 位置+引用 | ✅/⚠️/❌ | 全部含文件:行号 |
| AC2 SEV+置信度 | ✅/⚠️/❌ | 全部标注 |
| AC3 SEV1 ≤ 5 | ✅/⚠️/❌ | SEV1 = N |
| AC4 RC1-RC6 全覆盖 | ✅/⚠️/❌ | 6 维度均处理 |

## 禁用写法（出现即扣分）

| 禁用 | 替换 |
|---|---|
| "建议优化此处" | "建议提取为独立方法 cancelAndReleaseInventory()，理由：..." |
| "可能有并发问题" | "OrderService:88 update 后未加锁，并发取消会导致 release 调用两次。建议改用乐观锁版本号" |
| 所有问题 SEV 都是 "严重" | 按 SEV 标准严格分级 |
| SEV1 = 8 条 | 立即合并同类，或降级到 SEV2 |
| 同一类问题在 6 个位置各报一次 | 合并为 1 条 + "其他位置：file2:L10, file3:L20, ..." |
