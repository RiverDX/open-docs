# grill-deep-loop —— 深度提问（AI 执行版）

> **何时启动**：用户要"演练 review / 帮我想问题 / grill / 找问题 / 深度提问"。  
> **禁止**：空泛问题；问 PRD 已写的内容；同问题换皮。

---

## 输入清单（缺即停）

| 项 | 必需 | 缺失时动作 |
|---|---|---|
| 模块名 / 功能名 | 必需 | 反问 |
| 已有理解材料（reverse-loop 输出 / PRD / 模块文档之一） | 必需 | 反问，或建议先跑 reverse-loop |
| 关键代码 | 强烈建议 | 缺时问题不能含行号锚点 |
| 数据库 schema | 可选 | 缺时 GQ1 边界条件类问题质量会降 |

---

## 流程（6 步，禁止跳步）

1. **读**：通读已有理解材料 + 关键代码
2. **抽场景**：识别本功能的核心场景（用户路径、消息链、状态流转）
3. **按维度生成**：按 GQ1–GQ6 六维度逐项出问题，每维 ≥ 2 问
4. **填四字段**：每条问题填齐：问题 / 锚点 / 不答的后果 / 期望答案空间
5. **自查**：按 PASS 条件 GQ1–GQ8 填自查表
6. **终输出**：4 段（问题清单 + 自查表 + 优先级 5 问 + 业务方专属）

---

## PASS 条件（自查标准，严格解释）

### GQ1 边界条件维度（≥2 问）
- ✅ ≥ 2 问，覆盖 null / 空集合 / 最大值 / 最小值 / 越界 / 默认值 中至少 2 种
- ⚠️ 仅 1 问 或 2 问但都是 null 类
- ❌ 0 问

### GQ2 异常路径维度（≥2 问）
- ✅ ≥ 2 问，覆盖 超时 / 失败 / 第三方挂掉 / 输入畸形 / 鉴权失败 中至少 2 种
- ⚠️ 仅 1 问
- ❌ 0 问

### GQ3 事务与一致性（≥2 问）
- ✅ ≥ 2 问，含事务边界 + 跨库或分布式之一
- ⚠️ 仅 1 问
- ❌ 0 问

### GQ4 幂等与重试（≥2 问）
- ✅ ≥ 2 问，覆盖 重复请求 / 重试策略 / 回调多次 / 消息重复 / 唯一键冲突 中至少 2 种
- ⚠️ 仅 1 问
- ❌ 0 问

### GQ5 并发与时序（≥2 问）
- ✅ ≥ 2 问，覆盖 竞态 / 锁 / 状态机非法跳转 / 时序依赖 中至少 2 种
- ⚠️ 仅 1 问
- ❌ 0 问

### GQ6 缓存 / 历史数据（≥2 问）
- ✅ ≥ 2 问，覆盖 缓存与库一致 / 历史脏数据 / 字段语义演变 / 已废弃枚举 中至少 2 种
- ⚠️ 仅 1 问
- ❌ 0 问

### GQ7 每问都有具体锚点
- ✅ 100% 问题含 `文件名:行号` 或具体方法 / 字段 / 接口名
- ⚠️ ≥ 80% 有
- ❌ < 80%

### GQ8 每问都有"不答的后果"
- ✅ 100% 问题有非空"不答的后果"字段
- ⚠️ ≥ 80% 有
- ❌ < 80%

## 禁用问法（出现即扣分）

| 禁用问法 | 应改为 |
|---|---|
| "如何保证数据一致性？" | "OrderService.cancel():88 内先 update status 再调 InventoryClient，失败时是否 rollback？（不答的后果：卖超）" |
| "性能怎么样？" | "OrderMapper.listByStatus() 查询是否有 status 字段索引？（不答的后果：高峰期慢查询）" |
| "有没有考虑并发？" | "OrderTimeoutJob 多实例部署时如何防止同一订单被取消两次？（不答的后果：库存释放两次）" |
| "PRD 里说的 X 是什么意思？" | **删掉** —— PRD 已写的不要问 |

---

## 优先级 5 问的选择标准

按以下顺序选 5 条最有杀伤力的：

1. **会出生产事故**的（数据漂移、卖超、漏单、资损）
2. **架构级**的（分布式事务、消息丢失、状态机非法跳转）
3. **跨团队依赖**的（外部 client 行为约定、消息格式版本）
4. **业务规则歧义**的（业务方专属，但影响代码实现）
5. **回归风险高**的（改动会破坏已有功能）
